# Дата: 2026-02-25

Сегодня полностью сосредоточился на фундаменте проекта. Настроил базовую инфраструктуру Frontend-части, чтобы в дальнейшей разработке не возникало проблем с качеством кода и согласованностью коммитов.

Выбор инструментов:

 - В качестве пакетного менеджера Bun.
 - Основной стек: Vue 3 + TypeScript.
 - Контроль качества кода (ESLint & Prettier)

Что было сделано:

 - Внедрил ESlint (Flat Config). Настроил правила для JS/TS и Vue-шаблонов.
 - Подключил Perfectionist для автоматической сортировки импортов и свойств (чтобы в коде был порядок).
 - Настроил резолвинг импортов и проверку на неиспользуемые импорты.
 - Добавил нативную проверку CSS файлов.
 - Полностью интегрировал Prettier через eslint-config-prettier, чтобы избежать конфликтов форматирования.
 
 Автоматизация через Git Hooks (Husky):

 - Pre-commit: Теперь перед каждым коммитом автоматически запускается lint-staged. Он проверяет файлы (JS, TS, Vue, CSS)  - через ESLint и Prettier. Также запускается строгая проверка типов (vue-tsc). Если есть ошибки — коммит блокируется.
- Commit-msg: Внедрил стандарт Conventional Commits. Теперь сообщения должны иметь префиксы (feat, fix, chore и т.д.).

Скрипты:
Добавил удобные команды в package.json: lint, lint:fix, format, typecheck

✅ Итог:
Инфраструктура готова. Теперь проект защищен от «мусорного» кода и хаотичных коммитов. Локальная разработка настроена, можно приступать к функциональным задачам.

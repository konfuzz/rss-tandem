# Дата: 2026-03-23

Сегодня занимался стабилизацией backend-проверок после объединения frontend и backend в один репозиторий.

Что сделал:

- разделил CI на frontend и backend: добавил отдельный backend workflow и вынес frontend workflow в отдельный файл;
- добавил в backend недостающие скрипты для CI (`typecheck`, `ci:format`), чтобы backend проверялся теми же базовыми шагами, что и frontend;
- перевел Husky на один корневой `.husky`, чтобы hooks больше не конфликтовали между `frontend` и `backend`;
- настроил общий `pre-commit`, который последовательно запускает проверки frontend и backend, и привел в порядок `commit-msg` и `pre-push`;
- починил backend ESLint stack: добавил `eslint-import-resolver-typescript`, обновил `eslint-plugin-perfectionist`, исключил `dist` из lint и добавил root `.vscode/settings.json` для нормальной работы ESLint в VS Code из корня монорепозитория;
- привел backend-файлы к рабочему состоянию по стилю и форматированию, чтобы проект начал проходить `lint`, `typecheck`, `ci:format` и `build`.

По итогу:

- backend начал нормально участвовать в CI;
- локальные git hooks перестали конфликтовать;
- линтер подхватывается в VS Code при работе из корня репозитория;
- базовые backend-проверки теперь проходят стабильно и реально отражают состояние проекта.
